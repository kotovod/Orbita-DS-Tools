# Инструкции по тестированию плагина Orbita Icon Checker

Этот документ содержит инструкции по тестированию плагина Orbita Icon Checker для проверки его работоспособности и корректности проверок.

## Подготовка к тестированию

1. Установите плагин в Figma, следуя инструкциям в README.md
2. Создайте тестовый файл Figma или используйте существующий файл с иконками
3. Создайте несколько тестовых иконок с различными ошибками для проверки работы плагина

## Тестовые сценарии

### 1. Проверка корректной иконки

Создайте иконку, полностью соответствующую требованиям:

1. Создайте Component Set с именем "orb-icon-test"
2. Добавьте варианты с properties:
   - Variant: outline, solid
   - Size: lg, md, sm, xs, xxs
3. Для каждого варианта:
   - Установите правильный размер (32x32 для lg, 24x24 для md и т.д.)
   - Создайте слой Color-layer с выравниванием по центру
   - Внутри Color-layer создайте слой Vector и заблокируйте его
   - Добавьте скрытую группу Edit

Запустите плагин и проверьте, что для этой иконки не выводятся ошибки.

### 2. Проверка структуры

Создайте иконку с ошибками в структуре:

1. Создайте Component Set с именем "orb-icon-test-structure"
2. В одном из вариантов:
   - Не создавайте слой Color-layer
   - В другом варианте создайте Color-layer, но без Vector внутри

Запустите плагин и проверьте, что плагин обнаруживает эти ошибки.

### 3. Проверка размеров

Создайте иконку с ошибками в размерах:

1. Создайте Component Set с именем "orb-icon-test-size"
2. В одном из вариантов установите неправильный размер (например, 30x30 вместо 32x32)
3. В другом варианте не выравнивайте Color-layer по центру

Запустите плагин и проверьте, что плагин обнаруживает эти ошибки.

### 4. Проверка вариантов

Создайте иконку с ошибками в вариантах:

1. Создайте Component Set с именем "orb-icon-test-variants"
2. Создайте вариант без свойства Variant или Size
3. Создайте вариант с неправильным значением свойства (например, Variant=filled вместо solid)

Запустите плагин и проверьте, что плагин обнаруживает эти ошибки.

### 5. Проверка именования

Создайте иконку с ошибками в именовании:

1. Создайте Component Set с неправильным именем (например, "icon-test" вместо "orb-icon-test")
2. В одном из вариантов назовите слой неправильно (например, "color" вместо "Color-layer")

Запустите плагин и проверьте, что плагин обнаруживает эти ошибки.

### 6. Проверка цвета и стилей

Создайте иконку с ошибками в цвете:

1. Создайте Component Set с именем "orb-icon-test-color"
2. В одном из вариантов добавьте цвет непосредственно к слою Vector

Запустите плагин и проверьте, что плагин обнаруживает эти ошибки.

### 7. Проверка блокировки Vector

Создайте иконку, в которой Vector не заблокирован:

1. Создайте Component Set с именем "orb-icon-test-lock"
2. В одном из вариантов не блокируйте слой Vector

Запустите плагин и проверьте, что плагин обнаруживает эту ошибку.

### 8. Проверка группы Edit

Создайте иконку без группы Edit или с видимой группой Edit:

1. Создайте Component Set с именем "orb-icon-test-edit"
2. В одном из вариантов не создавайте группу Edit
3. В другом варианте создайте группу Edit, но сделайте её видимой

Запустите плагин и проверьте, что плагин обнаруживает эти ошибки.

## Проверка функциональности перехода к компоненту

1. Запустите плагин и выполните проверку иконок
2. В списке ошибок кликните на название компонента
3. Убедитесь, что Figma переходит к этому компоненту и выделяет его

## Отчет о тестировании

После выполнения всех тестовых сценариев, составьте отчет о тестировании, указав:

1. Какие проверки работают корректно
2. Какие проверки не работают или работают некорректно
3. Предложения по улучшению плагина

## Устранение проблем

Если во время тестирования возникают проблемы:

1. Проверьте консоль разработчика в Figma (Plugins → Development → Open Console)
2. Проверьте, что все файлы плагина (manifest.json, code.js, ui.html) содержат корректный код
3. Попробуйте перезапустить Figma и плагин
