## 🚀 Workflow

### Для публикации в Community:

```bash
# 1. Проверьте что используете правильный манифест
cat manifest.json | grep "AI Design Lint"
# Должно быть пусто - AI Design Lint отсутствует

# 2. Протестируйте локально с production манифестом
# Figma → Plugins → Development → Import manifest.json

# 3. Опубликуйте
# Figma → Plugins → Publish
```

### Для локальной разработки AI Design Lint:

```bash
# 1. Переключитесь на dev манифест
# Figma → Plugins → Development → Import manifest.dev.json

# 2. Теперь AI Design Lint доступен в меню!

# 3. Разрабатывайте и тестируйте
```

## 💡 Преимущества Такого Подхода

### ✅ Для вас как разработчика:
- Не нужно удалять/восстанавливать код
- Быстрое переключение между версиями
- Можно дорабатывать AI Design Lint параллельно
- Нет риска потерять код

### ✅ Для пользователей Community:
- Видят только стабильные функции
- Не запутаются в бета-функциях
- Плагин не требует AI токены (которых у них нет)

### ✅ Для безопасности:
- AI API endpoints не раскрываются публично
- Меньше surface area для атак
- Контролируемый доступ к экспериментальным функциям

## ❓ FAQ

**Q: Что если забуду какой манифест использую?**
A: Посмотрите в меню Figma - если есть AI Design Lint, значит dev версия.

**Q: Код AI Design Lint занимает место, это проблема?**
A: Нет, код не влияет на производительность если функция не вызывается.

**Q: Могу ли я позже вернуть AI Design Lint в production?**
A: Да! Просто добавьте обратно в `manifest.json` когда будет готов.

**Q: Нужно ли менять версию плагина?**
A: Да, при публикации обновите версию в manifest.json.
